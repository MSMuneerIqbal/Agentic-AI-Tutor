## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

The plan MUST demonstrate compliance with `docs/constitution.md` (Tutor GPT Constitution). Minimum gates:

- MCP Usage: All live search/docs/UI testing calls route via MCPs (TAVILY, Context7, Playwright). No ad‑hoc scraping or static crawling.
- Model Policy: All model calls use Gemini via `GEMINI_BASE_URL` and `GEMINI_API_KEY`.
- Backend‑First: Backend agent capabilities (Agents SDK + typed tools) precede frontend work; frontend depends on stable APIs.
- Prompt‑as‑Config: Prompts live under `app/agents_prompts/` in Six‑Part format, loaded at startup with hot‑reload in dev.
- Guardrails: Input/output/tripwire guardrails declared and tested. On trigger: sanitize response and log non‑sensitive event.
- Data Stores: Durable storage (MySQL/Postgres) for required entities; Redis for sessions/counters/cache with TTL; Pinecone for RAG with metadata‑only storage unless permitted.
- Observability: Structured JSON logs; metrics for latency, agent errors, guardrail triggers, and tool errors; alerting plan defined.
- CI & Quality: TDD approach with coverage targets and CI gates (lint, tests, coverage, Docker build). Offline CI mocks network calls.
- Security: Secrets from Render configuration only; never checked into code or logs.
